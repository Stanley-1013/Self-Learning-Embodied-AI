## Intuition

KaTeX compiles LaTeX-style source into styled HTML + MathML at build time, no runtime JS needed.

## Implementation Link

Every generated chapter relies on this pipeline — frontmatter metadata, KaTeX math, collapsible details for answers.

## Common Misconceptions

- KaTeX does not require runtime JS; only the stylesheet.
- `$` markers need to hug the math content on both sides; stray dollar signs in prose do not trigger parsing.

## Situational Questions

<details>
<summary>Q1: Does the summation render correctly? $\sum_{i=1}^{n} i = \frac{n(n+1)}{2}$</summary>

If you see the summation symbol and fraction, KaTeX is working.
</details>

<details>
<summary>Q2: Why KaTeX over MathJax?</summary>

- Faster rendering (synchronous, no reflow)
- Smaller bundle
- Sufficient coverage for robotics / ML notation
</details>

<details>
<summary>Q3: If a formula does not render, first triage step?</summary>

1. Check `docusaurus.config.ts` has `remark-math` and `rehype-katex` in `remarkPlugins` / `rehypePlugins`
2. Confirm `katex.min.css` is being loaded
3. Verify `$` pairs are balanced
</details>
