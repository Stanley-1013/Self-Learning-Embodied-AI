## Ch12 Dynamic Obstacle Avoidance

### Findings

- **[HIGH]** 12:707-708 — 機械臂 HRI 片段的零空間投影用錯投影子。C++ 代碼：
  ```cpp
  Eigen::MatrixXd N = I - J_pinv * J;  // null-space projector
  tau_null = N * tau_avoid;
  ```
  但 `tau_avoid` 是**關節力矩**（來自 $J_i^T F_{rep}$）。$(I - J^+ J)$ 是**速度/位置**的零空間投影子（用於 $\dot{q}$）；力矩零空間的正確投影子是 $(I - J^T (J^T)^+) = (I - J^T J^{+T})$（即本章 14-force-impedance-control.md 第 374-376 行的寫法，也是 Ch14 反覆強調的標準）。這兩個投影子**互為轉置**，在非方陣 $J$ 上**不等**。
  
  驗證：欲使 $\tau_{null}$ 不影響末端 wrench，需 $J^{+T} \tau_{null} = 0$。
  - 用 $(I - J^T J^{+T})$：$J^{+T} (I - J^T J^{+T}) \tau_{avoid} = J^{+T} \tau_{avoid} - (J^{+T} J^T) J^{+T} \tau_{avoid} = J^{+T} \tau_{avoid} - J^{+T} \tau_{avoid} = 0$ ✓
  - 用 $(I - J^+ J)$：$J^{+T} (I - J^+ J) \tau_{avoid} = J^{+T} \tau_{avoid} - J^{+T} J^+ J \tau_{avoid} \neq 0$ 一般而言
  
  結果：工人推手肘時，末端夾爪**並不是「紋絲不動」**，而是會有殘餘末端力 → 與 HRI 段落敘述（「末端夾爪紋絲不動」）自相矛盾。Category: 數學 / 演算法幻覺. Fix: 改為 `N = I - J.transpose() * J_pinv.transpose()`，與 Ch14 line 374 統一. Confidence: high.

- **[MED]** 12:75 & 12:224 — ORCA $\mathbf{n}$ 方向說明寫「$\mathbf{n}$ 取作 $\mathbf{u}$ 的單位向量（指向 VO 外側）」。這個簡化在**相對速度位於 VO 內部**（碰撞將發生）時正確，$u$ 與 $n$ 同向；但在**相對速度位於 VO 外**（當前路徑安全）時，$u$ 是從 $v_{rel}$ 朝最近邊界點的向量 → 指向 VO 內部；此時 $n$（VO 邊界外法向）會與 $u$ 反向。van den Berg 2011 原文 ORCA 演算法對這兩種情形有明確區分（見 paper Algorithm 1）。目前敘述把「一般情況」簡化為「$n = \hat u$」會讓讀者在讀原論文或自寫實作時遇到 sign flip 的困惑。Category: 概念方向 / 數學. Fix: 補一句「當 $v_{rel}$ 已在 VO 內時 $u$ 與 $n$ 同向；若在 VO 外則 $n = -\hat u$」，或直接改寫為「$n$ 取 VO 邊界在最近點處的**外法線**，方向永遠指向 VO 外側（與 $u$ 同向或反向視 $v_{rel}$ 位置而定）」. Confidence: high.

- **[LOW]** 12:91 — CBF 物理意義補述寫「$-\dot h$ 是實際靠近危險邊界的速率…向危險方向速度幾何級數衰減至零」。「幾何級數衰減」是針對 $\alpha(h) = \gamma h$ 的線性 class-K 函數成立（解為 $h(t) \geq h(0) e^{-\gamma t}$）；若用更一般的 class-K∞ 函數（如 $\alpha(h) = \gamma h^3$），衰減就不是幾何級數。一般 CBF 理論只保證「保持 $h \geq 0$」而非「指數衰減」。Category: 數學 / 術語精確性. Fix: 改為「若取 $\alpha(h) = \gamma h$ 則 $h$ 呈指數衰減；更一般的 class-K 函數只保證不越界」. Confidence: medium.

## Ch14 Force & Impedance Control

### Findings

- **[HIGH]** 14:274-276 — **Adjoint wrench transform 方向錯誤**（round-1 fix 引入的語意不一致）：文字寫 $W_A = \mathrm{Ad}_{T_{AB}}^T \cdot W_B$，但這與 Lynch & Park §3.4 Eq 3.96 / power-invariance duality 導出的結果相反。
  
  嚴格推導：給 $T_{AB} = (R, p)$ = 「B 在 A 中」，twist 變換 $V_A = \mathrm{Ad}_{T_{AB}} V_B$（章節寫對了）。由 power invariance $W_A^T V_A = W_B^T V_B$：
  $$W_A^T \mathrm{Ad}_{T_{AB}} V_B = W_B^T V_B \implies W_B = \mathrm{Ad}_{T_{AB}}^T W_A$$
  
  即 $W_A \to W_B$ 時乘 $\mathrm{Ad}_{T_{AB}}^T$，方向與章節敘述**相反**。等價寫法：$W_A = \mathrm{Ad}_{T_{BA}}^T W_B = \mathrm{Ad}_{T_{AB}^{-1}}^T W_B$。
  
  物理直觀驗證：純旋轉無平移時（$p=0$），$\mathrm{Ad}_T = \mathrm{diag}(R, R)$，$\mathrm{Ad}_T^T = \mathrm{diag}(R^T, R^T)$。物理力向量在 A 座標的分量 $f_A = R f_B$（$R$ 是 B → A 的旋轉）。若套章節公式 $W_A = \mathrm{Ad}_T^T W_B$：得到 $f_A = R^T f_B$ — 方向反了。正確應為 $W_B = \mathrm{Ad}_T^T W_A$（即 $f_B = R^T f_A$），或 $W_A = \mathrm{Ad}_{T^{-1}}^T W_B$.
  
  **但**：同段程式碼 `K_tcp = Ad_T.transpose() * K_flange * Ad_T`（line 303）**在語意正確**！因為：若 `T_flange_to_tcp` 解讀為 $T_{fl,tcp}$（tcp 在 flange 中），則 $V_{fl} = \mathrm{Ad}_{T_{fl,tcp}} V_{tcp}$ 且（由正確 dual）$W_{tcp} = \mathrm{Ad}_{T_{fl,tcp}}^T W_{fl}$，最後 $K_{tcp} = \mathrm{Ad}_{T_{fl,tcp}}^T K_{fl} \mathrm{Ad}_{T_{fl,tcp}}$ 符合程式碼。
  
  所以**程式碼對、文字公式方向錯** — 讀者若照文字套用其他座標轉換（例如法蘭盤 → base）會直接出錯。Category: 概念方向 / 數學. Fix: 把 line 274-276 改為：
  > $W_B = \mathrm{Ad}_{T_{AB}}^T \cdot W_A$（等價於 $W_A = \mathrm{Ad}_{T_{BA}}^T W_B$）
  
  並把 line 303 註解改為「$K_{tcp} = \mathrm{Ad}_{T_{fl,tcp}}^T K_{fl} \mathrm{Ad}_{T_{fl,tcp}}$（因 $W_{tcp} = \mathrm{Ad}_{T_{fl,tcp}}^T W_{fl}$）」以保持一致. Confidence: high（對照 Lynch & Park Eq 3.95-3.96 明確）.

- **[MED]** 14:280 — 「單純的力 $f$ 經過 $\hat{p} R \cdot f$ 這一塊會耦合出額外力矩 $m$」— 這敘述搭配「$\mathrm{Ad}_T = [R, 0; \hat p R, R]$」時，$\hat p R \cdot f_B$ 這塊乘的是 twist 的**線速度 $v_B$**，結果送去 $\omega$ 位置產生**角速度**分量（twist 變換），不是「力變成力矩」。如果要講「力產生力臂力矩」應該用 $\mathrm{Ad}_T^T = [R^T, -R^T \hat p; 0, R^T]$ 的右上 $-R^T \hat p$ 區塊作用在 $f$ 上得到角部分 — 這才是力臂耦合到力矩的物理解釋。目前「$\hat p R \cdot f$」的 narrative 跟 Ad_T 的**非轉置形式**對應，但該形式作用在 twist 上。Category: 數學 / 推理斷鏈. Fix: 改為「力經過 $\mathrm{Ad}_T^T$ 的右上 $-R^T \hat p$ 區塊會耦合出額外力矩」，或改講 twist 側的物理意義（「線速度 $v_B$ 經 $\hat p R \cdot v$ 耦合出角速度」）. Confidence: medium.

- **[LOW]** 14:298 — 程式碼註解寫「Ad_T = [R, 0; p̂R, R]（twist 伴隨矩陣；wrench 轉換用 Ad_T^T）」— 「wrench 轉換用 Ad_T^T」的表述本身沒錯（這是 dual 關係），但結合 line 274-276 的方向錯誤，整段的因果鏈斷裂。若按 HIGH 項修好 line 274-276，這裡自然一致. Confidence: low（相依於 HIGH 項）.
