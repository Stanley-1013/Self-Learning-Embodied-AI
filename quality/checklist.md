# 章節驗收檢查清單

每個章節在標記完成前必須通過以下檢查。`validate.sh` 會自動跑前 6 大項，人工再補後面 3 項。

---

## 自動檢查（由 `validate.sh` 執行）

### 1. Frontmatter 完整
- [ ] 檔案開頭有 `---` 包起來的 frontmatter
- [ ] 必要欄位：`title`、`prerequisites`、`estimated_time`、`difficulty`、`tags`

### 2. 必要段落齊全
章節必須依序包含以下 `##` 段落：
- [ ] 你將學到
- [ ] 核心概念
- [ ] 直覺理解
- [ ] 實作連結
- [ ] 常見誤解
- [ ] 練習題
- [ ] 面試角度
- [ ] 延伸閱讀

### 3. 練習題數量
- [ ] 至少 3 題，包在 `<details>` 摺疊區塊內，含解答

### 4. KaTeX 語法
- [ ] `$` 符號成對配對（奇數時警告，需人工確認）
- [ ] block 公式用 `$$...$$`，inline 用 `$...$`

---

## 人工檢查

### 5. 內容正確性
- [ ] 數學推導無錯誤（交叉比對來源教科書/論文）
- [ ] 符號慣例與前置章節一致
- [ ] 前置需求（`prerequisites`）真的都存在且已寫好

### 6. 教學品質
- [ ] 「直覺理解」段落用類比或視覺化降低抽象度
- [ ] 「常見誤解」列出 ≥2 個真實會被搞混的點
- [ ] 「實作連結」具體到可以想像程式碼長怎樣
- [ ] 面試角度聚焦「關鍵答題點」，不是整章摘要

### 7. 渲染檢查（啟動 `npm run dev`）
- [ ] KaTeX 公式在瀏覽器渲染正確
- [ ] 側邊欄導航出現這一章
- [ ] 前置章節連結可點擊
- [ ] 練習題 `<details>` 可展開/收合

---

## 失敗處理

- 自動檢查失敗 → 修到全綠再移到 `done.md`
- 人工檢查失敗 → 回去修，不要把問題留給「下次再說」
- 暫時無法修 → 在 `current.md` 標註 blocker，不要假裝完成
